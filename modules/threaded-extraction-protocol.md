# Threaded Extraction Protocol (Paced Mode)

---

## Purpose

Enable high-signal extraction from large threads without cognitive overload or scope drift.

---

## Core Principles

• Single-pass execution (one scope at a time)  
• Strict filtering (no cross-category leakage)  
• Sequential progression (no anticipation of future passes)  
• Copy-ready outputs only  

---

## Pass Structure

1. Architecture changes  
2. Execution / workflow changes  
3. Failure modes / constraints  
4. Language / framing  

---

## Execution Rules

• Only complete ONE pass at a time  
• Do not expand scope  
• Do not merge categories  
• Stop after each pass  

---

## Output Requirements

• Single clean block  
• Minimal structure  
• No meta-commentary  
• Match system voice  

---

## Use Case

Applied when extracting structured insight from:
• long ChatGPT threads  
• complex system discussions  
• multi-domain workflows  

---

## Result

Produces high-signal, reusable outputs suitable for:
• Evolution Thread integration  
• system refinement  
• module externalization
